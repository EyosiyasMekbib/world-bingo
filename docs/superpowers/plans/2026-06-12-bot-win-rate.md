# Configurable Bot Win Rate Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a per-template `botWinRate` (0–100%) setting, configurable in the admin dashboard, that controls how often bots claim bingo when their cartela wins — letting admins reduce bot wins in favor of human players.

**Architecture:** A new `botWinRate Int @default(100)` column on `GameTemplate`. When a bot's cartela first matches the winning pattern during a game, the bot service rolls once against the template's win rate. Pass → claim bingo immediately (current behavior). Fail → that bot is silenced for the rest of the game (it never claims, letting a human win). The roll happens **once per bot per game** — re-rolling on every ball would inflate the effective probability. Default of 100 preserves existing behavior for all current templates.

**Tech Stack:** Prisma 5 migration, Fastify route Zod schemas, Vitest unit tests, Nuxt 3 + @nuxt/ui admin form.

**Key behavior decision:** The win rate is a *ceiling on claiming*, not a guarantee of winning. Bots can never win more than their random cards allow. A silenced bot stays silent even if its card keeps matching on later balls.

import { describe, it, expect } from 'vitest'
import { BotService } from '../services/bot.service'

describe('BotService.shouldBotClaim', () => {
    it('always claims at 100% regardless of roll', () => {
        expect(BotService.shouldBotClaim(100, () => 0.999)).toBe(true)
        expect(BotService.shouldBotClaim(100, () => 0)).toBe(true)
    })

        expect(BotService.shouldBotClaim(0, () => 0.001)).toBe(false)

        expect(BotService.shouldBotClaim(40, () => 0.39)).toBe(true)
        expect(BotService.shouldBotClaim(40, () => 0.40)).toBe(false)
        expect(BotService.shouldBotClaim(40, () => 0.99)).toBe(false)

    it('treats null/undefined rate as 100 (legacy templates)', () => {
        expect(BotService.shouldBotClaim(null, () => 0.999)).toBe(true)
        expect(BotService.shouldBotClaim(undefined, () => 0.999)).toBe(true)
    })

    it('clamps out-of-range rates', () => {
        expect(BotService.shouldBotClaim(150, () => 0.999)).toBe(true)
        expect(BotService.shouldBotClaim(-10, () => 0)).toBe(false)
    })
})

- [ ] **Step 3: Implement `shouldBotClaim`**

In `apps/api/src/services/bot.service.ts`, add inside the `BotService` class (above `playBotTurns`):

```typescript
    /**
     * Decide whether a bot should claim a winning bingo, based on the
     * template's botWinRate (0–100). null/undefined means 100 (always claim),
     * which preserves behavior for templates created before this setting existed.
     * `rng` returns a float in [0, 1) and is injectable for deterministic tests.
     */
    static shouldBotClaim(winRate: number | null | undefined, rng: () => number = Math.random): boolean {
        const rate = Math.min(100, Math.max(0, winRate ?? 100))
        if (rate >= 100) return true
        if (rate <= 0) return false
        return rng() * 100 < rate
    }
```

### Task 3: Wire the roll into `playBotTurns`

**Files:**
- Modify: `apps/api/src/services/bot.service.ts` (`playBotTurns`, currently lines ~160–244)

Two changes: load `botWinRate` with the game, and roll once per bot when its cartela first matches. A failed roll silences that bot for the rest of the game.

- [ ] **Step 1: Load the template's win rate**

In `playBotTurns`, the existing pattern load is:

```typescript
            // Load game pattern (stable for the lifetime of the game)
            const initialGame = await prisma.game.findUnique({
                where: { id: gameId },
                select: { pattern: true },
            })
            if (!initialGame) return
            const pattern = initialGame.pattern as PatternName
```

Replace with:

```typescript
            // Load game pattern and bot win rate (stable for the lifetime of the game)
            const initialGame = await prisma.game.findUnique({
                where: { id: gameId },
                select: { pattern: true, template: { select: { botWinRate: true } } },
            })
            if (!initialGame) return
            const pattern = initialGame.pattern as PatternName
            const botWinRate = initialGame.template?.botWinRate

            // Bots that matched the pattern but lost the win-rate roll stay
            // silent for the rest of the game — rolling again on every new
            // ball would inflate the effective claim probability.
            const silencedBotIds = new Set<string>()
```
